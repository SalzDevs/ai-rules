### Selected Rules
- react.no-direct-fetch

### Mandatory Instructions
- [react.no-direct-fetch] Do not call `fetch` directly inside React components. Extract data fetching into custom hooks. Prefer: Use a custom hook (e.g. `useUsers`, `usePosts`) to encapsulate fetch logic, state management, and error handling.

### Examples
Rule react.no-direct-fetch:
```tsx
// Bad — fetch inside a component
function UsersPage() {
  const [users, setUsers] = useState([]);
  useEffect(() => {
    fetch("/api/users").then((r) => r.json()).then(setUsers);
  }, []);
  return <UserList users={users} />;
}

// Good — extracted into a custom hook
function UsersPage() {
  const { data: users, isLoading } = useUsers();
  return <UserList users={users} isLoading={isLoading} />;
}
```
